# Claude Code チーム構成（サブエージェント一式）

Webアプリケーション開発プロジェクト向けに、役割ごとのClaude Codeサブエージェントを定義したフォルダです。

## 使い方

1. `.claude` フォルダを、開発対象プロジェクトのルート直下にそのまま配置してください。
   ```
   your-project/
   ├── .claude/
   │   └── agents/
   │       ├── tech-lead.md
   │       ├── product-owner.md
   │       ├── pm.md
   │       ├── frontend-dev.md
   │       ├── backend-dev.md
   │       ├── qa-infra.md
   │       └── ui-designer.md
   ├── src/
   └── ...
   ```
2. そのプロジェクトで `claude` コマンドを起動すると、`.claude/agents/` 内のエージェントが自動的に読み込まれます。
3. 明示的に呼び出したい場合は「frontend-devサブエージェントを使ってログイン画面を実装して」のように依頼してください。内容が合致すれば、Claude Codeが自動的に委任することもあります。
4. `/agents` コマンドで、登録されているサブエージェントの一覧確認や編集もできます。

## 構成一覧

| ファイル | 役割 | 実装作業を任せられるか |
|---|---|---|
| tech-lead.md | テックリード（設計・技術選定・レビュー） | 提案・レビューまで |
| product-owner.md | プロダクトオーナー補佐（要件・仕様整理） | たたき台作成まで |
| pm.md | PM補佐（タスク分解・進捗管理） | 整理・要約まで |
| frontend-dev.md | フロントエンドエンジニア | ◎ 実装を任せられる |
| backend-dev.md | バックエンドエンジニア | ◎ 実装を任せられる |
| qa-infra.md | QA・インフラ担当 | ◎ 実装を任せられる |
| ui-designer.md | UI/UXデザイン担当（マークアップベース） | マークアップまで |

## 注意点

- `tech-lead` / `product-owner` / `pm` / `ui-designer` の4役は、判断材料の整理・たたき台作成までを担当する設計にしています。最終的な意思決定や対外調整（他部署・顧客との交渉、リリース判断など）は人間が行ってください。
- `frontend-dev` / `backend-dev` / `qa-infra` の3役は、実装作業をある程度任せられます。ただし本番環境への破壊的操作は各エージェントの制約により、実行前に人間の確認を求めるようにしています。
- 各ファイルの `tools` 項目でアクセス可能なツールを制限しています。プロジェクトの実態に合わせて調整してください。
- 各エージェントのプロンプトは叩き台です。実際のプロジェクトの規約・技術スタックに合わせて加筆修正することをおすすめします。
